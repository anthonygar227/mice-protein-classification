# Predicting Mouse Experimental Group from Protein Expression
## What this project set out to answer
This project uses the UCI Mice Protein Expression dataset, which records the expression levels of 77 proteins measured in the cerebral cortex of 1080 tissue samples. Every sample belongs to one of eight experimental groups. Those groups are built from three factors. The first is genotype, meaning a mouse is either a normal control or a trisomic model of Down syndrome. The second is treatment, meaning the mouse received either the drug memantine or a saline placebo. The third is behavior, meaning the mouse went through one of two learning tasks. The central question was whether the protein measurements alone carry enough information to recover which group a mouse belongs to, and if so, which proteins carry that signal. The deeper goal was never the accuracy number by itself. It was to learn whether protein expression encodes the underlying biological condition, and to do the analysis honestly enough that the result could be trusted.
## How the data was prepared
The first decision was to look at the data before changing anything. That inspection revealed three things that shaped everything afterward. The label column was named in lowercase, three of the eighty columns were text descriptors rather than protein measurements, and the eight groups were roughly balanced in size. Looking first is not a formality. It is the habit that prevents writing code that quietly does the wrong thing.
Cleaning had two parts. The three text columns named Genotype, Treatment, and Behavior were removed because those three values combined are exactly what defines the group label. Leaving them in would have handed the model pieces of the answer, which is a form of cheating that produces a meaningless score. The second part was handling missing values. The data contained 1396 empty cells spread unevenly across the proteins. A few proteins were missing a quarter or more of their values, while most were missing only a handful. The rule applied was to drop any protein missing more than fifteen percent of its values. The reasoning is that filling a few gaps with a sensible estimate is fair, but filling a column that is mostly empty means inventing a large fraction of the data, which would mislead the model rather than help it. Five proteins crossed that threshold and were removed, leaving 72.
## The methodology that protects the result
The single most important idea in the whole project is the prevention of data leakage. Leakage is when information from the held-out test data influences the training process, which makes the final score look better than the model truly deserves. The defense was to split the data into a training set and a test set first, and then to learn every data-dependent transformation from the training set alone. Imputation learned its median fill values from training only. Scaling learned its averages and spreads from training only. Feature selection scored and chose the top thirty proteins using training only. In each case the test set merely received the already learned settings. This discipline, fitting the training and only transforming the test set, was applied at three separate stages.
## The result that mattered most
The first version of the project used a random split, where individual rows were shuffled into training and test sets. That version reached 98.6 percent accuracy. A score that high invites suspicion rather than celebration, and the suspicion turned out to be justified. The dataset contains fifteen repeated measurements of each individual mouse. With a random split, several measurements of the same mouse could land in both the training and the test set. The model could then partly recognize a mouse it had already studied rather than learn the biology that generalizes to new mice. That is a subtler kind of leakage than leaving the answer in the features, but it inflates the score in the same way.
The fix was to split by mouse, so that every measurement of any given mouse stays entirely on one side of the split. After that change, no mouse appeared in both sets, and the honest accuracy fell from 98.6 percent to about 65 percent. That drop of roughly thirty-three points is the most valuable finding in the project. It quantifies how much of the original score was the leak rather than real learning. The honest number is still far above the chance baseline, which means protein expression genuinely does encode the experimental group, just far less perfectly than the inflated number implied.
## What the model relied on, and the limits of that
The final step asked which proteins the model leaned on, measured by how much accuracy dropped when each protein's values were randomly scrambled. The proteins at the top were ones already known to be biologically relevant to the condition being studied. It is reassuring that these remained at the top even after the leakage was removed, because it means the signal survived the honest test.
It is important to state clearly what this does not show. The model demonstrates that these proteins are useful for predicting the group. It does not show that they cause anything. The importance of a predictive model is a statement about correlation and usefulness, not about biological mechanism. Proving causation would require controlled intervention, which a model trained on existing measurements cannot provide. The right way to describe the feature importance result is that the model's most useful proteins line up with known biology, which builds confidence that the model latched onto a real signal rather than noise.
## What I would do next
The held-out test split and the cross-validation are both grouped by mouse, so every reported estimate is computed in the same honest way. Natural next refinements would be to compare the random forest against other model families, tune how many proteins to keep rather than fixing it at thirty, and report confidence intervals on the honest accuracy. None of these would change the core lesson of the project, which is that a careful split and a willingness to report a less flattering but truer number is worth more than a high score taken at face value.
## How to run it
The script downloads the dataset automatically from the UCI Machine Learning Repository, so no data file is needed.
First install the required packages.
pip install ucimlrepo scikit-learn pandas numpy
Then run the script.
python mice_protein_rf.py
The script prints the baseline accuracy, the cross validated accuracy, the held out test accuracy, a per class performance report, and the ranked list of the proteins the model relied on most.
